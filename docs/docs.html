Các trường hợp sử dụng quy trình AI (AI Workflow Use Cases)

Ngữ cảnh: Bạn đang làm dự án FE viết note, giống như Notion

7 kịch bản: 5 kịch bản dùng bộ skill /morkit:*, cộng 2 kịch bản dựng UI từ Figma (qua Figma MCP hoặc file HTML tự export). Thể hiện rõ từng bước phía dưới, kèm output dự kiến của AI. Mỗi Use Case (UC) được chia thành các bước (step) có Tại sao · Bạn làm · Bạn thấy, để người mới follow được.

Dùng sidebar bên trái chuyển UC, hoặc click vào từng bước (step) để chuyển hướng nhanh.

Brownfield onboarding — từ partial inputs đến bộ docs team-verified

Tình huống thực tế: bạn vừa được giao một repo brownfield (đã có code, đôi khi có vài tài liệu rời rạc như README, brainstorm, design mockup) nhưng thiếu SRS / API docs / DB design chính thức. UC này hướng dẫn cách dùng /morkit:init để dựng bộ docs lần đầu, rồi dùng /morkit:update-doc để duy trì theo feedback của team. Demo trên repo Notion-clone (Next.js 15 · TipTap · Drizzle · Supabase).

Đọc theo thứ tự — mỗi bước (step) có 3 phần: Tại sao bước này? · Bạn làm gì? · Bạn thấy gì? Người mới nên chú ý BƯỚC 4 (plan-approval gate) — đây là chỗ quyết định chất lượng cả bộ docs.

input: partial docs + source 6 bước 2 human gates diff engine preserve edits ~30 phút realtime · ~120k tokens
