AI Workflow Use Cases

5 kịch bản dùng /morkit:* trên repo Notion-clone, mô phỏng qua giao diện CLI giả định. Mỗi UC tách thành các step có Tại sao · Bạn làm · Bạn thấy để người mới follow được.

Dùng sidebar bên trái chuyển UC, hoặc click step để nhảy thẳng.

Brownfield onboarding — từ partial inputs đến bộ docs team-verified

Tình huống thực tế: bạn vừa được giao một repo brownfield (đã có code, đôi khi có vài tài liệu rời rạc như README, brainstorm, design mockup) nhưng thiếu SRS / API docs / DB design chính thức. UC này hướng dẫn cách dùng /morkit:init để dựng bộ docs lần đầu, rồi dùng /morkit:update-doc để duy trì theo feedback của team. Demo trên repo Notion-clone (Next.js 15 · TipTap · Drizzle · Supabase).

Đọc theo thứ tự — mỗi step có 3 phần: Tại sao bước này? · Bạn làm gì? · Bạn thấy gì? Người mới nên chú ý STEP 4 (plan-approval gate) — đây là chỗ quyết định chất lượng cả bộ docs.

input: partial docs + source 6 step 2 human gates diff engine preserve edits ~30 phút realtime · ~120k tokens

Greenfield project — từ ý tưởng đến bộ docs full team-verified

Tình huống thực tế: bạn được giao 1 dự án greenfield — chưa có code, chưa có docs, chỉ có idea note / requirement / market research. UC này hướng dẫn 7 step từ brainstorm ý tưởng → function list → SRS & screen-spec (tầng business) → API/DB/Arch (tầng technical / HLD) → team sign-off. Cách tiếp cận top-down 2 vòng: chốt business trước, chốt technical sau — tránh tốn công khi business chưa rõ.

Khác UC1 thế nào? UC1 (brownfield) có code & partial docs làm anchor → /morkit:init 1 lần ra hết. UC2 (greenfield) không có anchor → phải brainstorm trước, tách /morkit:init làm 2 vòng (vòng 1: SRS · vòng 2: HLD) để dừng lại cho PM/BrSE/BA review giữa chừng.

input: idea / requirement only 7 step 3 human gates 2 vòng /morkit:init ~2-3 ngày realtime · ~250k tokens
