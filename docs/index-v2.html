/plugin add marketplace github:mor-duongmh/claude-plugins
/plugin install morkit@mor-duongmh

2. morkit có những gì?

Một plugin gói 4 nhóm chức năng, tất cả gọi qua tiền tố /morkit:*:

NhómBao gồmĐể làm gì
Spec workflow /morkit:propose, /morkit:review, /morkit:archive Tự sinh proposal, design, tasks và checklist để bạn duyệt trước khi cho code chạy.
Plan & build brainstorming, writing-plans, executing-plans, subagent-driven-development, test-driven-development, systematic-debugging Suy nghĩ ý tưởng, viết plan, chạy plan từng bước, viết test trước, debug có hệ thống.
Code review /morkit:deep-review, /morkit:deep-review-doctor, /morkit:deep-review-post Review code chuyên sâu bằng 5 agent AI chạy song song.
Doc generation /morkit:setup, /morkit:init, /morkit:update-doc, /morkit:sync, /morkit:apply-sync, /morkit:doctor Tự sinh tài liệu dự án theo chuẩn ITO Japan offshore: SRS, API, schema DB, kiến trúc...

Tổng cộng 26 skill + 9 agent chuyên trách + 15 slash command, tất cả đều có tiền tố /morkit:.

3. Danh sách command

Bấm vào tên command để xem giải thích chi tiết, cách gọi và ví dụ.

Spec workflow

CommandĐể làm gì
/morkit:propose [mô tả]Sinh đầy đủ proposal, design, tasks và checklist trong một lần chạy.
/morkit:review [tên]Tạo hoặc làm mới checklist duyệt thiết kế cho một change.
/morkit:archive [tên]Đóng một change folder sau khi đã merge và deploy ổn.

Plan & build

SkillĐể làm gì
/morkit:brainstormingExplore mode — thinking partner free-form, không code; auto-save design log cuối phiên.
/morkit:writing-plansViết plan nhiều bước từ ý tưởng đã chốt.
/morkit:executing-plansChạy plan từng bước (bị review-gate chặn cho tới khi human duyệt).
/morkit:subagent-driven-developmentChạy plan song song bằng nhiều subagent — nhanh hơn executing tuần tự.
/morkit:test-driven-developmentTDD discipline — viết test trước, code sau, refactor cuối.
/morkit:systematic-debuggingDebug 5 bước có hệ thống — không đoán mò.
/morkit:dispatching-parallel-agentsTách 2+ task độc lập chạy song song bằng subagent.
/morkit:using-git-worktreesIsolate feature work bằng worktree riêng, tránh xung đột workspace.

Code review

CommandĐể làm gì
/morkit:deep-review [target]Review chuyên sâu trên PR hoặc git diff (5 agent AI chạy song song).
/morkit:deep-review-doctorKiểm tra cài đặt Deep Review đã đủ điều kiện chạy chưa.
/morkit:deep-review-postPost báo cáo review lên PR làm comment (không request changes).

Doc generation

CommandĐể làm gì
/morkit:setupDựng môi trường Python cho docs-hero (khoảng 30-60 giây, chạy 1 lần).
/morkit:initSinh bộ tài liệu mới (SRS, API, DB, arch...) từ ProjectModel JSON.
/morkit:update-docÁp dụng change hoặc plan đã chốt vào tài liệu hiện có (diff engine preserve manual edits).
/morkit:syncĐọc mã nguồn và đề xuất nội dung nên cập nhật vào tài liệu (read-only).
/morkit:apply-syncÁp dụng các nội dung bạn đã tick trong sync-proposal.md.
/morkit:doctorKiểm tra cài đặt docs-hero xem có ổn không.

4. Agent chuyên trách

9 agent được orchestrator gọi theo nhu cầu — không phải command bạn gõ trực tiếp.

Deep review (7 specialist)

risk-impact-analyst
Tính blast radius + risk score cho diff dùng code-review-graph.
security-auditor
OWASP Top 10 · secret leakage · unsafe deserialization · SSRF · injection.
pattern-architecture-critic
Design pattern violations · anti-pattern · architectural-boundary breaches.
test-coverage-auditor
Function mới/sửa chưa có test · missing test case qua tests_for relation.
convention-checker
Naming · structure · style — ưu tiên CLAUDE.md hơn language profile.
performance-auditor
Perf smell: N+1 · sync I/O hot path · unbounded loop · alloc tight loop.
documentation-auditor
Docstring/comment · README update · migration notes cho public API thay đổi.

Doc generation + core

docs-hero
QA sau generate-srs/api/db — cross-ref FR↔Screen↔API↔TBL, BrSE quality, Mermaid syntax. Read-only.
code-reviewer
Reviewer chính khi cần đánh giá code thuần (ngoài deep-review pipeline).

5. Companion tools

Hai công cụ giúp agent trả lời chính xác hơn và tiết kiệm token. Plugin không cài lặng lẽ — sẽ hỏi bạn trước.

Công cụVai tròCách cài
Context7 Trả về tài liệu/API đúng phiên bản cho thư viện, agent không phải đoán. Cài lười — plugin tự gọi npx -y ctx7 khi cần.
RTK Nén output của lệnh bash, giảm 60-90% token. Hỏi 1 lần ở phiên đầu — bạn chọn Cài / Bỏ qua / Đừng hỏi lại.
Context7 đã được bật sẵn trong 6 skill nhóm Plan & build và 3 skill nhóm Spec workflow — agent sẽ tự gọi Context7 thay vì đoán API.

Cài RTK thủ công

curl -fsSL https://raw.githubusercontent.com/rtk-ai/rtk/refs/heads/master/install.sh | sh
rtk init -g

Cài Context7 dạng MCP (đầy đủ tính năng)

npx -y ctx7 setup
